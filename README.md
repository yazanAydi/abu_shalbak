# أبو شلبك — نظام إدارة السوبرماركت

نظام عربي متكامل لنقطة البيع وإدارة المتجر. بديل حديث لنظام حساباتي يعمل على شبكة المحل المحلية (LAN).

## المتطلبات

- **Docker + Docker Compose** (موصى به للإنتاج)
- أو: **Node.js ≥ 18** للتشغيل المباشر

---

## تشغيل المتجر (Docker — الإنتاج)

خادم واحد يشغّل كل شيء: API + قاعدة البيانات + لوحة الإدارة + نقطة البيع + Telegram + النسخ الاحتياطي.

### 1. تثبيت Docker على جهاز الخادم (PC المتجر)

- ثبّت [Docker Desktop](https://www.docker.com/products/docker-desktop/) (Windows) أو Docker Engine (Linux)
- عيّن **IP ثابت** للجهاز، مثلاً: `192.168.1.10`

### 2. إعداد المشروع

```bash
cd abo_shalbak

# انسخ ملف البيئة وعدّل JWT_SECRET
copy .env.example .env    # Windows
# cp .env.example .env      # Linux/Mac
```

عدّل `.env` على الأقل:

```env
NODE_ENV=production
PORT=3000
HOST=0.0.0.0
JWT_SECRET=ضع-هنا-سر-عشوائي-طويل
DATABASE_PATH=/app/data/supermarket.db
BACKUP_DIR=/app/backups
```

### 3. تشغيل

```bash
docker compose up -d
```

### 4. فتح التطبيق من أي جهاز في الشبكة

| التطبيق | الرابط | الجهاز |
|---------|--------|--------|
| **لوحة الإدارة** | `http://192.168.1.10:3000/admin` | مدير / محاسب |
| **نقطة البيع** | `http://192.168.1.10:3000/pos` | كاشير |
| **API** | `http://192.168.1.10:3000/api` | — |

> استبدل `192.168.1.10` بـ IP جهاز الخادم الفعلي.

### 5. اختصارات سطح المكتب (اختياري)

على كل جهاز، أنشئ اختصار Chrome/Edge:

- **إدارة:** `http://192.168.1.10:3000/admin`
- **كاشير:** `http://192.168.1.10:3000/pos`

**أجهزة الكاشير والإدارة تحتاج فقط متصفح Chrome أو Edge — لا حاجة لتثبيت Node أو React.**

### 6. البيانات والنسخ الاحتياطي

| المجلد | المحتوى |
|--------|---------|
| `./data/` | قاعدة SQLite (`supermarket.db`) — **تبقى بعد إعادة تشغيل Docker** |
| `./backups/` | نسخ احتياطي تلقائي يومياً الساعة 2:00 صباحاً |

```bash
# نسخ يدوي
docker compose exec -T app cp /app/data/supermarket.db /app/backups/manual_$(date +%Y%m%d).db

# أو انسخ مجلد ./data كاملاً إلى USB أو سحابة
```

### 7. إعادة البناء بعد تحديث الكود

```bash
docker compose down
docker compose build --no-cache
docker compose up -d
```

## تشغيل بدون Docker (تطوير)

```bash
# مرة واحدة
npm install

# شغّل الخادم والواجهة معاً
npm run dev
#  خادم API: http://localhost:5000
#  لوحة الإدارة: http://localhost:3001
#  نقطة البيع: http://localhost:3002
```

### متغيرات البيئة

أنشئ `.env` في جذر المشروع و `frontend/.env`:

```
# .env (backend)
PORT=5000
JWT_SECRET=change-me-to-long-random-string
DATABASE_PATH=./data/supermarket.db
HOST=0.0.0.0
```

```
# frontend/.env (لوحة الإدارة — المنفذ 3001)
PORT=3001
REACT_APP_POS_URL=http://127.0.0.1:3002

# frontend-pos/.env (نقطة البيع — المنفذ 3002)
PORT=3002
REACT_APP_ADMIN_URL=http://127.0.0.1:3001
```

### تطبيقان منفصلان (إدارة + كاشير)

| التطبيق | المجلد | المنفذ | الاستخدام |
|---------|--------|--------|-----------|
| **لوحة الإدارة** | `frontend/` | 3001 | تقارير، منتجات، عملاء، بنوك، إعدادات — **بدون كاشير** |
| **نقطة البيع** | `frontend-pos/` | 3002 | كاشير، ورديات، بيع، إرجاع — **بدون شاشات الإدارة** |

تشغيل الكل معاً من جذر المشروع:

```bash
npm run install:all   # أول مرة فقط
npm start             # API :5000 + إدارة :3001 + POS :3002
```

أو بشكل منفصل: `npm run start:admin` و `npm run start:pos` مع `npm run start:api`.

## استيراد المنتجات من حساباتي

### تصدير من حساباتي
1. ادخل إلى **بطاقة الأصناف** → **تصدير إلى Excel**
2. احفظ الملف `.xlsx`

### استيراد في أبو شلبك
1. سجّل دخول كـ **مدير**
2. اذهب إلى **إدارة المنتجات** → قسم **رفع منتجات**
3. ارفع ملف Excel

الأعمدة المدعومة تلقائياً (عربي وإنجليزي):

| عربي | إنجليزي |
|------|---------|
| باركود | barcode |
| الاسم / اسم المنتج | name |
| السعر / سعر البيع | price |
| التكلفة / سعر الشراء | cost |
| المخزون / الكمية | stock |
| التصنيف / القسم | category |
| الضريبة / نسبة الضريبة | tax |
| الوحدة | unit |
| تاريخ الصلاحية | expiry_date |

## النسخ الاحتياطي

النسخ الاحتياطي التلقائي يعمل يومياً في الإنتاج (Docker). راجع قسم **تشغيل المتجر** أعلاه.

```bash
# نسخ يدوي (Docker)
docker compose exec -T app cp /app/data/supermarket.db /app/backups/manual_$(date +%Y%m%d).db

# أو انسخ مجلد ./data كاملاً
cp -r ./data ./backup/data_$(date +%Y%m%d)
```

## خطة الانتقال من حساباتي

| المرحلة | المدة | الهدف |
|---------|-------|-------|
| **0 — تجريبي** | أسبوع-أسبوعين | تشغيل أبو شلبك بجانب حساباتي؛ استيراد المنتجات |
| **1 — البيع والضريبة** | 2-3 أسابيع | الكاشير والورديات وضريبة القيمة المضافة |
| **2 — المخزون والتقارير** | 2-3 أسابيع | الجرد والصلاحية وتقارير المبيعات |
| **3 — العملاء** | أسبوعان | بطاقة العميل والذمم |
| **4 — المحاسبة الخفيفة** | 4-5 أسابيع | البنوك والشيكات وسندات القبض والصرف |

**قاعدة:** أبقِ حساباتي مفتوحاً للقراءة فقط حتى يوقّع المحاسب على تقارير المرحلة 4.
