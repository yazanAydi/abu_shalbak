## 1. ما الذي ستثبّته؟

```
جهاز خادم واحد (PC المتجر)
    │
    ├── قاعدة البيانات (SQLite)
    ├── لوحة الإدارة     →  /admin   (مدير / محاسب)
    ├── نقطة البيع POS   →  /pos     (كاشير)
    └── نسخ احتياطي تلقائي يومياً

أجهزة الكاشير والمكتب → متصفح Chrome أو Edge فقط
(لا Docker، لا Node.js، لا تثبيت إضافي)
```

**الفكرة:** جهاز واحد يشغّل النظام. باقي الأجهزة تفتح رابط في المتصفح فقط.

---

## 2. ماذا تحتاج قبل البدء؟

| العنصر | التفاصيل |
|--------|----------|
| **جهاز الخادم** | PC يبقى مشغّلاً أثناء ساعات العمل |
| **أجهزة الكاشير** | أي PC بنظام Windows + Chrome أو Edge |
| **جهاز المكتب** | للمدير أو المحاسب — متصفح فقط |
| **الشبكة** | كل الأجهزة على **نفس Wi‑Fi / LAN** |
| **الإنترنت** | اختياري (مطلوب فقط لـ Telegram) |

> **مهم:** في كل هذا الدليل، الرقم `192.168.1.10` **مثال فقط**.  
> استبدله دائماً بـ **IP جهازك الحقيقي** (تتعلّمه في [القسم 4](#4-كيف-تجد-ip-جهاز-الخادم)).

## 4. كيف تجد IP جهاز الخادم

IP هو عنوان جهازك على شبكة المحل. بدونه لا يستطيع الكاشير الوصول للنظام.

### الطريقة 1 — من إعدادات Windows (الأسهل)

1. على **جهاز الخادم**، اضغط زر **Start** (ابدأ).
2. افتح **Settings** (الإعدادات — أيقونة الترس).
3. اذهب إلى **Network & Internet** (الشبكة والإنترنت).
4. اضغط **Wi‑Fi** أو **Ethernet** — حسب اتصال الجهاز.
5. اضغط على اسم الشبكة المتصلة.
6. مرّر لأسفل إلى **Properties** (الخصائص).
7. ابحث عن **IPv4 address** — مثل: `192.168.1.15`
8. **اكتب هذا الرقم** على ورقة أو في ملف نصي.

```
مثال:
IPv4 address: 192.168.1.15   ← هذا IP جهازك
```

### الطريقة 2 — من سطر الأوامر

1. اضغط **Windows + R**.
2. اكتب `cmd` واضغط **Enter**.
3. اكتب `ipconfig` واضغط **Enter**.
4. ابحث عن قسم **Wi‑Fi** أو **Ethernet** النشط.
5. اقرأ **IPv4 Address**.

```
مثال على الناتج:
Wireless LAN adapter Wi-Fi:
   IPv4 Address. . . . . . . . . . . : 192.168.1.15
```

### كيف تستخدم IP في الروابط

| مثال خاطئ (لا تستخدمه) | رابطك الصحيح (إذا IP = 192.168.1.15) |
|------------------------|--------------------------------------|
| `http://192.168.1.10:3000/admin` | `http://192.168.1.15:3000/admin` |
| `http://192.168.1.10:3000/pos` | `http://192.168.1.15:3000/pos` |

- Docker يستخدم المنفذ **3000**
- Node.js بدون Docker يستخدم المنفذ **5000**

## 5. كيف تجعل IP ثابتاً

إذا تغيّر IP بعد إعادة التشغيل، اختصارات الكاشير ستتوقف عن العمل.  
**افعل هذا مرة واحدة** على جهاز الخادم.

### الخيار أ — من الراوتر (الأفضل)

1. افتح المتصفح واكتب عنوان الراوتر — غالباً `192.168.1.1` أو `192.168.0.1`.
2. سجّل دخول (اسم المستخدم وكلمة المرور على ملصق الراوتر).
3. ابحث عن: **DHCP Reservation** أو **Static IP** أو **Address Reservation**.
4. أضف **MAC address** لجهاز الخادم وعيّن IP ثابت (مثل `192.168.1.15`).
5. احفظ.

**لإيجاد MAC address:**
Settings → Network → اتصالك → **Physical address (MAC)**

### الخيار ب — يدوياً في Windows

1. Settings → **Network & Internet** → **Wi‑Fi** أو **Ethernet**.
2. اضغط شبكتك → **Edit** بجانب IP assignment.
3. غيّر من **Automatic (DHCP)** إلى **Manual**.
4. فعّل **IPv4** وأدخل:

| الحقل | القيمة النموذجية |
|-------|-----------------|
| IP address | `192.168.1.15` (اختر رقماً غير مستخدم) |
| Subnet mask | `255.255.255.0` |
| Gateway | `192.168.1.1` (IP الراوتر) |
| DNS | `192.168.1.1` أو `8.8.8.8` |

5. احفظ.

## 6. التثبيت الرئيسي — Docker (موصى به)

اتبع هذه الخطوات **بالترتيب** على جهاز الخادم.

---

### الخطوة 6.1 — انسخ المشروع إلى جهاز الخادم

1. انسخ مجلد `abo_shalbak` كاملاً إلى جهاز الخادم.
2. المسار المقترح:

```
C:\abo_shalbak
```

---

### الخطوة 6.2 — ثبّت Docker Desktop

1. حمّل من: [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop/)
2. شغّل المثبّت واتبع التعليمات.
3. أعد تشغيل الجهاز إذا طُلب منك.
4. افتح **Docker Desktop** وانتظر حتى يظهر **Running** (أخضر).

**تحقق من التثبيت:**

1. اضغط **Start** واكتب `PowerShell` واضغط **Enter** (نافذة عادية — ليس Administrator).
2. اكتب:

```powershell
docker --version
docker compose version
```

3. يجب أن يظهر رقم إصدار لكل أمر. إذا ظهر خطأ → Docker غير مشغّل، افتح Docker Desktop.

---

### الخطوة 6.3 — أنشئ ملف `.env`

**ما هو `.env`؟** ملف إعدادات يحتوي كلمة سر النظام ومسار قاعدة البيانات.

1. افتح **PowerShell** (عادي):

```powershell
cd C:\abo_shalbak
copy .env.example .env
```

2. افتح الملف في Notepad:

```powershell
notepad .env
```

3. عدّل هذه الأسطر (أو أضفها إن لم تكن موجودة):

```env
NODE_ENV=production
PORT=3000
HOST=0.0.0.0

JWT_SECRET=ضع-هنا-سر-عشوائي-طويل

DATABASE_PATH=/app/data/supermarket.db
BACKUP_DIR=/app/backups
```

4. **ولّد سر JWT** — في PowerShell:

```powershell
[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 }))
```

5. انسخ الناتج (سلسلة طويلة عشوائية) والصقه مكان `ضع-هنا-سر-عشوائي-طويل`.
6. في Notepad: **File → Save** ثم أغلق Notepad.

> **تحذير:** لا تترك `JWT_SECRET` بالقيمة الافتراضية `change-me...` — الخادم قد يرفض التشغيل في الإنتاج.

---

### الخطوة 6.4 — شغّل النظام

في PowerShell (عادي):

```powershell
cd C:\abo_shalbak
docker compose up -d
```

**أول مرة تستغرق 5–15 دقيقة** (تحميل + بناء واجهة الإدارة ونقطة البيع).

**تحقق أن الخادم يعمل:**

```powershell
docker compose ps
```

يجب أن ترى الحالة **running**.

**شاهد السجل (اختياري):**

```powershell
docker compose logs -f app
```

اضغط `Ctrl+C` للخروج من السجل (الخادم يبقى يعمل).

**ما يجب أن تراه في السجل:**
- Server listening on port **3000**
- `[db] Using database file: /app/data/supermarket.db`

> قاعدة البيانات تُنشأ **تلقائياً** في أول تشغيل مع مستخدمين افتراضيين.

---

### الخطوة 6.5 — افتح PowerShell كمسؤول (Administrator)

تحتاج صلاحيات مسؤول لفتح المنفذ في جدار الحماية.

1. اضغط **Start**.
2. اكتب: `PowerShell`
3. **لا تضغط Enter مباشرة.**
4. **انقر بالزر الأيمن** على **Windows PowerShell**.
5. اختر **Run as administrator** (تشغيل كمسؤول).
6. إذا ظهرت نافذة "Do you want to allow...?" → اضغط **Yes**.
7. تأكد أن عنوان النافذة يقول: **Administrator: Windows PowerShell**

---

### الخطوة 6.6 — افتح المنفذ 3000 في جدار الحماية

**لماذا؟** Windows يمنع الأجهزة الأخرى (الكاشير) من الاتصال بالخادم إلا إذا سمحت بذلك.

في **PowerShell كمسؤول**، انسخ هذا السطر كاملاً واضغط **Enter**:

```powershell
New-NetFirewallRule -DisplayName "Abo Shalbak POS" -Direction Inbound -Protocol TCP -LocalPort 3000 -Action Allow
```

| جزء الأمر | المعنى |
|-----------|--------|
| `New-NetFirewallRule` | إنشاء قاعدة جديدة |
| `-DisplayName "Abo Shalbak POS"` | اسم القاعدة |
| `-Direction Inbound` | اتصالات **الواردة** إلى هذا الجهاز |
| `-Protocol TCP` | نوع الاتصال |
| `-LocalPort 3000` | المنفذ الذي يستخدمه Docker |
| `-Action Allow` | السماح (لا الحظر) |

**نجاح:** يظهر ناتج فيه `Enabled: True` و `Action: Allow` — بدون رسائل حمراء.

**فشل "Access is denied":** أغلق PowerShell وكرر [الخطوة 6.5](#الخطوة-65--افتح-powershell-كمسؤول-administrator).

**تحقق يدوي (اختياري):**
Start → **Windows Defender Firewall** → **Advanced settings** → **Inbound Rules** → ابحث عن **Abo Shalbak POS** ✓

---

### الخطوة 6.7 — اختبر أن الصفحتين تعملان

**"إذا تحمّلت الصفحتان → الخادم جاهز"**

#### أ) تأكد أن Docker يعمل

```powershell
docker compose ps
```

#### ب) افتح صفحة الإدارة (Admin)

1. على **جهاز الخادم**، افتح **Google Chrome** أو **Microsoft Edge**.
2. اضغط على **شريط العنوان** (أعلى المتصفح حيث تكتب الروابط).
3. اكتب (استبدل IP برقمك من [القسم 4](#4-كيف-تجد-ip-جهاز-الخادم)):

```
http://192.168.1.15:3000/admin
```

4. اضغط **Enter**.

| النتيجة | ماذا تعني |
|---------|-----------|
| ✅ صفحة تسجيل دخول عربية (اسم مستخدم + كلمة مرور) | **نجاح** |
| ❌ "This site can't be reached" | الخادم غير مشغّل — ارجع للخطوة 6.4 |
| ❌ "ERR_CONNECTION_REFUSED" | نفس الحل — أو IP خاطئ |
| ❌ صفحة بيضاء فارغة لأكثر من 30 ثانية | انتظر أو أعد بناء Docker |

#### ج) افتح صفحة نقطة البيع (POS)

1. اضغط **Ctrl + T** (تبويب جديد).
2. اكتب:

```
http://192.168.1.15:3000/pos
```

3. اضغط **Enter**.

| النتيجة | ماذا تعني |
|---------|-----------|
| ✅ صفحة تسجيل دخول الكاشير | **نجاح** |

#### د) اختبر من جهاز آخر (كاشير أو مكتب)

على **PC مختلف** على **نفس Wi‑Fi**، افتح نفس الرابطين.

| الفحص | ✓ |
|-------|---|
| Admin يفتح على جهاز الخادم | ☐ |
| POS يفتح على جهاز الخادم | ☐ |
| Admin يفتح من جهاز كاشير/مكتب | ☐ |
| POS يفتح من جهاز كاشير/مكتب | ☐ |
| تسجيل دخول `admin` / `admin123` يعمل | ☐ |

**كلها ✓ → الخادم جاهز للخطوات التالية.**

**يعمل على الخادم لكن لا يعمل على أجهزة أخرى؟** → كرر [الخطوتين 6.5 و 6.6](#الخطوة-65--افتح-powershell-كمسؤول-administrator) (جدار الحماية).

## 7. إعداد أجهزة الكاشير والمكتب

على **كل** جهاز كاشير أو مكتب — **لا تثبّت Docker ولا Node.js**.

### الخطوة 7.1 — اتصل بنفس الشبكة

تأكد أن الجهاز على **نفس Wi‑Fi / LAN** مثل جهاز الخادم.

### الخطوة 7.2 — ثبّت المتصفح

ثبّت **Google Chrome** أو **Microsoft Edge** إن لم يكن موجوداً.

### الخطوة 7.3 — أنشئ اختصار الكاشير

1. انقر بالزر الأيمن على **Desktop** (سطح المكتب) → **New** → **Shortcut**.
2. في "Type the location of the item" اكتب (استبدل IP):

```
http://192.168.1.15:3000/pos
```

3. **Next** → الاسم: `كاشير - أبو شلبك` → **Finish**.
4. انقر مرتين على الاختصار → يجب أن تظهر صفحة تسجيل الدخول.

### الخطوة 7.4 — أنشئ اختصار الإدارة (جهاز المكتب)

نفس الخطوات، لكن الرابط:

```
http://192.168.1.15:3000/admin
```

الاسم: `إدارة - أبو شلبك`

### الخطوة 7.5 — (اختياري) ملء الشاشة للكاشير

في Chrome/Edge اضغط **F11** لوضع ملء الشاشة.

## 8. أول تسجيل دخول واستيراد المنتجات

### الخطوة 8.1 — حسابات الدخول الافتراضية

| اسم المستخدم | كلمة المرور | الدور |
|--------------|-------------|-------|
| `admin` | `admin123` | مدير (صلاحيات كاملة) |
| `cashier1` | `cashier123` | كاشير |

1. افتح **Admin** → سجّل دخول: `admin` / `admin123`
2. **غيّر كلمة مرور المدير فوراً** (قد يُجبرك النظام على ذلك)
3. أنشئ حسابات كاشير حقيقية
4. عطّل الحسابات التجريبية التي لا تحتاجها

### الخطوة 8.2 — استيراد المنتجات من حساباتي

#### من حساباتي (تصدير)

1. افتح حساباتي → **بطاقة الأصناف**
2. **تصدير إلى Excel**
3. احفظ ملف `.xlsx`

#### إلى أبو شلبك (استيراد)

1. Admin → **إدارة المنتجات** → **رفع منتجات**
2. ارفع ملف Excel
3. تحقق من عينة: باركود، سعر، مخزون

**الأعمدة المدعومة:**

| عربي | English |
|------|---------|
| باركود | barcode |
| الاسم / اسم المنتج | name |
| السعر / سعر البيع | price |
| التكلفة / سعر الشراء | cost |
| المخزون / الكمية | stock |
| التصنيف / القسم | category |
| الضريبة | tax |
| الوحدة | unit |
| تاريخ الصلاحية | expiry_date |

### الخطوة 8.2.1 — استيراد ملفات حساباتي الأخرى (ترحيل البيانات)

بعد **بطاقة الأصناف**، يمكن رفع ملفات Excel الإضافية من حساباتي. يُكتشف نوع كل ملف تلقائياً من **اسم الملف** أو **عناوين الأعمدة**.

**الترتيب الموصى به:**

| # | ملف حساباتي | أين ترفعه في أبو شلبك |
|---|-------------|------------------------|
| 1 | بطاقة الأصناف | Admin → **إدارة المنتجات** → رفع منتجات |
| 2 | قائمة الأسعار | Admin → **إدارة المنتجات** → رفع منتجات |
| 3 | أرصدة الموردين | Admin → **إدارة الموردين** → استيراد أرصدة (`POST /api/admin/suppliers/upload`) |
| 4 | أرصدة زبون | Admin → **إدارة العملاء** → استيراد أرصدة (`POST /api/admin/customers/upload`) |
| 5 | أرصدة العمارة / المشغلين | Admin → **إدارة العملاء** → استيراد أرصدة |

**ملاحظات:**

- استخدم ملفات `.xlsx` حقيقية — **اختصارات Windows (`.lnk`) لا تعمل**.
- **أرصدة العمارة** و**المشغلين** تُستورد كبطاقات عملاء (فئة «شركات» أو وسم في الملاحظات).
- **قائمة الأسعار** تحدّث أسعار المنتجات الموجودة (مفرق → سعر البيع، جملة → سعر أدنى اختياري).
- إذا كان للعميل/المورد حركات محاسبية في أبو شلبك، لن يُستبدل رصيده إلا بمعامل `force=1` (نافذة الترحيل فقط).

### الخطوة 8.3 — إعدادات المتجر

في لوحة الإدارة، اضبط:

1. اسم المتجر وإعدادات ضريبة القيمة المضافة
2. طابعة الإيصالات (إن وُجدت)
3. أزرار البيع السريع للمنتجات الشائعة
4. أدوار المستخدمين (كاشير، محاسب، …)

## 9. اختبار أول عملية بيع

### على جهاز الكاشير

1. افتح رابط POS → سجّل دخول ككاشير
2. **افتح وردية** — أدخل مبلغ الصندوق الافتتاحي
3. امسح باركود أو اختر منتجاً
4. أكمل **عملية بيع تجريبية** (نقداً)
5. اطبع أو اعرض الإيصال

### على جهاز الإدارة

1. Admin → التقارير → مبيعات اليوم
2. تأكد أن المبلغ يطابق عملية البيع التجريبية

**إذا تطابق → النظام جاهز للتشغيل.**

## 10. النسخ الاحتياطي والتشغيل اليومي

### أين تُحفظ البيانات

| المجلد | المحتوى |
|--------|---------|
| `C:\abo_shalbak\data\` | قاعدة البيانات الحية `supermarket.db` |
| `C:\abo_shalbak\backups\` | نسخ احتياطي **تلقائي يومياً الساعة 2:00 ص** |

### نسخ احتياطي يدوي

```powershell
cd C:\abo_shalbak
docker compose exec -T app cp /app/data/supermarket.db /app/backups/manual_backup.db
```

أو انسخ مجلد `data` إلى USB:

```powershell
xcopy C:\abo_shalbak\data D:\backup\data /E /I
```

| المهمة | الأمر |

## 11. التثبيت البديل — بدون Docker

استخدم هذه الطريقة إذا **لا تريد Docker**، أو تريد **إدارة عن بُعد عبر Tailscale**.

### 11.1 — ثبّت Node.js

1. حمّل [Node.js 18+ LTS](https://nodejs.org/)
2. ثبّت بالإعدادات الافتراضية
3. تحقق:

```powershell
node --version
npm --version
```

### 11.2 — ثبّت الحزم (مرة واحدة)

```powershell
cd C:\abo_shalbak
npm run install:all
```

### 11.3 — أنشئ `.env`

```powershell
copy .env.example .env
notepad .env
```

**للمحل على LAN (كل الأجهزة على نفس الشبكة):**

```env
NODE_ENV=production
PORT=5000
HOST=0.0.0.0
JWT_SECRET=<سر-عشوائي-طويل>
DATABASE_PATH=./data/supermarket.db
BACKUP_DIR=./backups
```

**لـ Tailscale فقط (الخادم غير مكشوف على LAN):**

```env
NODE_ENV=production
PORT=5000
HOST=127.0.0.1
JWT_SECRET=<سر-عشوائي-طويل>
DATABASE_PATH=./data/supermarket.db
```

ولّد JWT:

```powershell
[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 }))
```

### 11.4 — شغّل الخادم

```powershell
cd C:\abo_shalbak
npm run start:prod
```

**اترك نافذة PowerShell مفتوحة** — إغلاقها يوقف الخادم.

| التطبيق | الرابط (LAN) |
|---------|--------------|
| Admin | `http://YOUR_IP:5000/admin` |
| POS | `http://YOUR_IP:5000/pos` |

### 11.5 — جدار الحماية (منفذ 5000)

PowerShell **كمسؤول** ([الخطوة 6.5](#الخطوة-65--افتح-powershell-كمسؤول-administrator)):

```powershell
New-NetFirewallRule -DisplayName "Abo Shalbak POS" -Direction Inbound -Protocol TCP -LocalPort 5000 -Action Allow
```

اختبر الصفحتين كما في [الخطوة 6.7](#الخطوة-67--اختبر-أن-الصفحتين-تعملان) لكن استخدم **5000** بدل **3000**.

### 11.6 — تشغيل تلقائي عند بدء Windows (اختياري)

1. افتح **Task Scheduler**
2. Create Task → Trigger: **At log on**
3. Action → Start a program:
   - Program: `powershell.exe`
   - Arguments: `-NoExit -Command "cd C:\abo_shalbak; npm run start:prod"`
4. Save

## 12. اختياري — Telegram و Tailscale

### Telegram (موافقة الإرجاع + تنبيهات الصلاحية)

أضف في `.env`:

```env
TELEGRAM_USE_POLLING=1
TELEGRAM_REFUND_BOT_TOKEN=<من BotFather>
TELEGRAM_REFUND_CHAT_ID=<معرف مجموعة الإدارة>
TELEGRAM_EXPIRY_BOT_TOKEN=<بوت ثاني>
TELEGRAM_EXPIRY_CHAT_ID=<مجموعة تنبيهات الصلاحية>
```

1. أنشئ البوتات عبر [@BotFather](https://t.me/BotFather)
2. أضف البوتات لمجموعة المدير
3. أعد تشغيل الخادم
4. اختبر: كاشير يطلب إرجاع → يظهر زر موافقة في Telegram

### Tailscale (إدارة من الهاتف)

1. ثبّت [Tailscale](https://tailscale.com/download) على PC المحل والهاتف
2. سجّل دخول بنفس الحساب
3. شغّل الخادم بـ `HOST=127.0.0.1` و `npm run start:prod`
4. PowerShell **كمسؤول**:

```powershell
tailscale serve --bg 5000
```

5. افتح رابط HTTPS الذي يعطيك إياه Tailscale على الهاتف
6. سجّل دخول Admin

> POS يبقى محلياً في المحل فقط — Tailscale للإدارة عن بُعد.

التفاصيل: [DEPLOY-TAILSCALE.md](./DEPLOY-TAILSCALE.md)

## 13. حل المشاكل

| المشكلة | الحل |
|---------|------|
| "This site can't be reached" | الخادم متوقف → `docker compose up -d` أو `npm run start:prod` |
| يعمل على الخادم، لا يعمل على الكاشير | جدار الحماية → [6.5–6.6](#الخطوة-65--افتح-powershell-كمسؤول-administrator) |
| صفحة قديمة / خاطئة | **Ctrl + F5** (تحديث قوي) |
| IP توقف عن العمل بعد إعادة التشغيل | IP ثابت → [القسم 5](#5-كيف-تجعل-ip-ثابتا) |
| "Access is denied" في PowerShell | يجب **Run as administrator** |
| Docker = 3000، Node = 5000 | لا تخلط المنافذ |
| الخادم لا يبدأ | `JWT_SECRET` ضعيف أو افتراضي — غيّره |
| صفحة بيضاء | `docker compose build --no-cache` ثم `docker compose up -d` |
| بيانات خاطئة | تأكد من ملف DB واحد فقط: `data/supermarket.db` |
